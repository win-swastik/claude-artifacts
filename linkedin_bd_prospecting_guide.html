Step 1

Export Your LinkedIn Connections

LinkedIn's native export gives you every connection reliably — no scraping, no cookies, no success rate risk. This is always your starting point.

  1. Go to LinkedIn Settings
    Navigate to Me → Settings & Privacy → Data privacy → Get a copy of your data
  2. Request "Connections" data only
    Select the "Connections" checkbox (not the full archive — that takes days). Click Request archive.
  3. Download within 10 minutes
    LinkedIn emails you a link. Download and unzip. You'll find Connections.csv.
  4. Check the CSV columns
    The file has: First Name, Last Name, Email Address, Company, Position, Connected On, and crucially — Profile URL (the LinkedIn profile link for each person).
💡

If Profile URL is missing in the CSV, run the prepare script below — it builds URLs from the public identifier (last segment of each person's LinkedIn URL) which LinkedIn sometimes exports separately.

Step 2

Prepare Profile URLs from the CSV

This Python script reads your LinkedIn Connections.csv export, extracts profile URLs, and produces a clean JSON file of URLs ready to feed into Apify.

Python — prepare_urls.py
import csv
import json
import re
import sys

# ─── CONFIG ──────────────────────────────────────────────────────────────
INPUT_CSV    = "Connections.csv"   # path to your LinkedIn export
OUTPUT_JSON  = "profile_urls.json"  # will be created by this script
# ─────────────────────────────────────────────────────────────────────────

def clean_url(raw: str) -> str:
    """Normalise a LinkedIn profile URL."""
    raw = raw.strip()
    if not raw:
        return ""
    # Already a full URL
    if raw.startswith("http"):
        # strip query strings
        return raw.split("?")[0].rstrip("/")
    # Public identifier only (e.g. "john-doe-12345")
    if not raw.startswith("/") and not raw.startswith("linkedin"):
        return f"https://www.linkedin.com/in/{raw}"
    return f"https://www.linkedin.com{raw}"

def main():
    urls = []
    skipped = 0

    with open(INPUT_CSV, encoding="utf-8-sig") as f:
        reader = csv.DictReader(f)
        for row in reader:
            # LinkedIn may use different column names
            url_raw = (
                row.get("Profile URL", "") or
                row.get("URL", "") or
                row.get("linkedin_url", "") or
                row.get("LinkedIn Profile", "")
            )
            url = clean_url(url_raw)
            if url and "linkedin.com/in/" in url:
                urls.append(url)
            else:
                skipped += 1

    with open(OUTPUT_JSON, "w") as f:
        json.dump(urls, f, indent=2)

    print(f"✅  Extracted {len(urls)} profile URLs → {OUTPUT_JSON}")
    if skipped:
        print(f"⚠️   Skipped {skipped} rows (no URL found) — check column names in CSV")

if __name__ == "__main__":
    main()
💡

Run: python3 prepare_urls.py — it outputs profile_urls.json containing a plain array of LinkedIn URLs.

Step 3

Run the Apify Enrichment Actor

Feed your profile URLs into harvestapi/linkedin-profile-scraper to get full work histories, locations, and company data for every contact.

Option A — Apify Console (no-code)

  1. Go to apify.com/harvestapi/linkedin-profile-scraper and click Try for free.
  2. Set Mode to "Profile details no email ($4 per 1k)" — sufficient for DB alumni + PE/VC detection.
  3. Paste URLs — open profile_urls.json, copy the array, and paste it into the Target URLs field.
  4. Run and wait. At 100% success rate, all profiles will complete. Download results as JSON (not CSV — JSON preserves the nested work history array).

Option B — Apify API (programmatic)

Python — run_apify.py
import json, time, requests

APIFY_TOKEN = "YOUR_APIFY_API_TOKEN"   # from apify.com/account/integrations
ACTOR_ID    = "harvestapi/linkedin-profile-scraper"
BATCH_SIZE  = 500   # keep batches small to avoid timeouts

with open("profile_urls.json") as f:
    all_urls = json.load(f)

# Process in batches
all_results = []
for i in range(0, len(all_urls), BATCH_SIZE):
    batch = all_urls[i:i+BATCH_SIZE]
    print(f"Running batch {i//BATCH_SIZE + 1} ({len(batch)} profiles)...")

    # Start the actor run
    run_resp = requests.post(
        f"https://api.apify.com/v2/acts/{ACTOR_ID}/runs",
        params={"token": APIFY_TOKEN},
        json={
            "profileScraperMode": "Profile details no email ($4 per 1k)",
            "queries": batch
        }
    ).json()
    run_id = run_resp["data"]["id"]

    # Poll until finished
    while True:
        status = requests.get(
            f"https://api.apify.com/v2/actor-runs/{run_id}",
            params={"token": APIFY_TOKEN}
        ).json()["data"]["status"]
        if status in ("SUCCEEDED", "FAILED", "ABORTED"):
            break
        time.sleep(5)

    # Fetch results from dataset
    dataset_id = run_resp["data"]["defaultDatasetId"]
    items = requests.get(
        f"https://api.apify.com/v2/datasets/{dataset_id}/items",
        params={"token": APIFY_TOKEN, "format": "json"}
    ).json()
    all_results.extend(items)
    print(f"  → Retrieved {len(items)} profiles")

with open("enriched_profiles.json", "w") as f:
    json.dump(all_results, f, indent=2)

print(f"\n✅ Total enriched: {len(all_results)} profiles → enriched_profiles.json")

Step 4

Filter Script — Deutsche Bank Alumni & PE/VC Outside India

This is the core filtering engine. It reads the enriched Apify JSON and applies two targeting rules with geographic exclusion.

🎯 Targeting Rules

Bucket A — Deutsche Bank Alumni: Any contact who has Deutsche Bank (or DB variants) in their work history AND is currently located outside India.

Bucket B — PE/VC Professionals: Any contact whose current company or role title contains PE/VC keywords AND is located outside India.

Python — filter_contacts.py
import json
import csv
import re
from typing import Any

# ─── CONFIG ──────────────────────────────────────────────────────────────
INPUT_JSON       = "enriched_profiles.json"
OUTPUT_CSV_DB    = "db_alumni_outside_india.csv"
OUTPUT_CSV_PEVC  = "pevc_outside_india.csv"
OUTPUT_CSV_ALL   = "all_targets.csv"
# ─────────────────────────────────────────────────────────────────────────

# Deutsche Bank name variants (case-insensitive)
DB_PATTERNS = [
    r"\bdeutsche\s*bank\b",
    r"\bdb\s+ag\b",
    r"\bdeutsche\s*bank\s*ag\b",
    r"\bdb\s+london\b",
    r"\bdb\s+group\b",
    r"\bdwpbank\b",      # Deutsche WertpapierService Bank
]

# PE/VC keywords — company name OR job title
PEVC_COMPANY_PATTERNS = [
    r"\bprivate\s+equity\b",
    r"\bventure\s+capital\b",
    r"\bventure\s+partners\b",
    r"\bgrowth\s+equity\b",
    r"\bgrowth\s+capital\b",
    r"\bbuyout\b",
    r"\bleveraged\s+buyout\b",
    r"\bpe\s+fund\b",
    r"\bvc\s+fund\b",
    r"\bfund\s+of\s+funds\b",
    r"\bportfolio\s+company\b",
    r"\bmanagement\s+buyout\b",
]
PEVC_TITLE_PATTERNS = [
    r"\bprivate\s+equity\b",
    r"\bventure\s+capital\b",
    r"\bventure\s+partner\b",
    r"\binvestment\s+manager\b",
    r"\bportfolio\s+manager\b",
    r"\bgp\s+partner\b",
    r"\blimited\s+partner\b",
    r"\bfund\s+manager\b",
    r"\bprincipal\b",          # common PE title
    r"\bassociate\s+director\b",  # PE associate
    r"\bmanaging\s+director\b",
]
# Known PE/VC firm name fragments
PEVC_FIRM_NAMES = [
    "blackstone", "kkr", "carlyle", "apollo", "warburg pincus",
    "advent international", "general atlantic", "tpg", "bain capital",
    "sequoia", "andreessen horowitz", "a16z", "accel", "tiger global",
    "softbank vision", "insight partners", "summit partners",
    "permira", "cinven", "apax", "bc partners", "cvc capital",
    "vista equity", "francisco partners", "silver lake", "thoma bravo",
    "coatue", "dragoneer", "dsg consumer", "nexus ventures",
    "matrix partners", "lightspeed", "bessemer", "index ventures",
]

# India location markers (exclude these)
INDIA_MARKERS = [
    "india", "mumbai", "delhi", "bangalore", "bengaluru", "hyderabad",
    "pune", "chennai", "kolkata", "ahmedabad", "gurgaon", "gurugram",
    "noida", "chandigarh", "jaipur", "kochi", "in"  # country code "IN"
]


def matches_any(text: str, patterns: list) -> bool:
    text = (text or "").lower()
    return any(re.search(p, text, re.I) for p in patterns)

def is_india(profile: dict) -> bool:
    # Check all location fields the actor may return
    location_fields = [
        profile.get("location", ""),
        profile.get("locationName", ""),
        profile.get("geoLocationName", ""),
        profile.get("countryCode", ""),
        profile.get("country", ""),
        profile.get("city", ""),
    ]
    combined = " ".join(str(x) for x in location_fields if x).lower()
    return any(marker in combined for marker in INDIA_MARKERS)

def get_location(profile: dict) -> str:
    return (
        profile.get("location") or
        profile.get("locationName") or
        profile.get("geoLocationName") or
        "Unknown"
    )

def get_work_history(profile: dict) -> list:
    # harvestapi returns positions in various keys
    return (
        profile.get("positions") or
        profile.get("experience") or
        profile.get("workHistory") or
        []
    )

def is_db_alumni(profile: dict) -> tuple[bool, str]:
    """Returns (is_alumni, db_role_summary)."""
    positions = get_work_history(profile)
    db_roles = []
    for pos in positions:
        company = pos.get("companyName", "") or pos.get("company", "")
        title   = pos.get("title", "") or pos.get("role", "")
        if matches_any(company, DB_PATTERNS):
            db_roles.append(f"{title} @ {company}")
    return (bool(db_roles), " | ".join(db_roles))

def is_pevc(profile: dict) -> tuple[bool, str]:
    """Returns (is_pevc, reason)."""
    positions = get_work_history(profile)
    reasons = []
    # Check current position first
    current = positions[0] if positions else {}
    cur_company = (current.get("companyName", "") or current.get("company", "")).lower()
    cur_title   = (current.get("title", "") or current.get("role", "")).lower()
    headline    = (profile.get("headline", "") or "").lower()

    if matches_any(cur_company, PEVC_COMPANY_PATTERNS):
        reasons.append(f"Company: {current.get('companyName','')}")
    if any(firm in cur_company for firm in PEVC_FIRM_NAMES):
        reasons.append(f"Known firm: {current.get('companyName','')}")
    if matches_any(cur_title, PEVC_TITLE_PATTERNS):
        reasons.append(f"Title: {current.get('title','')}")
    if matches_any(headline, PEVC_COMPANY_PATTERNS + PEVC_TITLE_PATTERNS):
        reasons.append(f"Headline: {profile.get('headline','')}")

    return (bool(reasons), " | ".join(reasons))

def build_row(profile: dict, bucket: str, match_reason: str) -> dict:
    positions = get_work_history(profile)
    current = positions[0] if positions else {}
    return {
        "Bucket":          bucket,
        "First Name":      profile.get("firstName", ""),
        "Last Name":       profile.get("lastName", ""),
        "Headline":        profile.get("headline", ""),
        "Current Title":   current.get("title", "") or current.get("role", ""),
        "Current Company": current.get("companyName", "") or current.get("company", ""),
        "Location":        get_location(profile),
        "Profile URL":     profile.get("linkedInUrl", "") or profile.get("url", ""),
        "Match Reason":    match_reason,
        "Connections":     profile.get("connectionsCount", ""),
    }

def write_csv(rows: list, path: str):
    if not rows:
        print(f"  ⚠️  No results for {path}")
        return
    with open(path, "w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=rows[0].keys())
        writer.writeheader()
        writer.writerows(rows)
    print(f"  ✅  {len(rows)} contacts → {path}")


def main():
    with open(INPUT_JSON) as f:
        profiles = json.load(f)

    print(f"\n🔍 Processing {len(profiles)} enriched profiles...\n")

    db_rows, pevc_rows, all_rows = [], [], []
    stats = {"total": len(profiles), "india_skipped": 0, "no_location": 0}

    for p in profiles:
        location = get_location(p)

        # Skip if no location data (can't confirm outside India)
        if location == "Unknown":
            stats["no_location"] += 1
            continue

        # Exclude India-based contacts
        if is_india(p):
            stats["india_skipped"] += 1
            continue

        db_match, db_reason   = is_db_alumni(p)
        pv_match, pv_reason   = is_pevc(p)

        if db_match:
            row = build_row(p, "Deutsche Bank Alumni", db_reason)
            db_rows.append(row)
            all_rows.append(row)

        if pv_match:
            row = build_row(p, "PE/VC Professional", pv_reason)
            pevc_rows.append(row)
            if not db_match:   # avoid duplicates in combined list
                all_rows.append(row)

    # Write outputs
    write_csv(db_rows,   OUTPUT_CSV_DB)
    write_csv(pevc_rows, OUTPUT_CSV_PEVC)
    write_csv(all_rows,  OUTPUT_CSV_ALL)

    print(f"""
📊 Summary
──────────────────────────────
Total profiles processed : {stats['total']}
India-based (excluded)   : {stats['india_skipped']}
No location data         : {stats['no_location']}
Deutsche Bank alumni     : {len(db_rows)}
PE/VC professionals      : {len(pevc_rows)}
Combined unique targets  : {len(all_rows)}
──────────────────────────────
""")

if __name__ == "__main__":
    main()
💡

Run: python3 filter_contacts.py — produces three CSVs: DB alumni, PE/VC, and a merged all-targets file.

Troubleshooting

Common Issues & Fixes

IssueCauseFix
Filter finds 0 Deutsche Bank alumni Work history key name differs in actor output Print profiles[0].keys() and update the get_work_history() fallback keys
All locations show "Unknown" Actor returns location under a different field name Print profiles[0] to see the actual JSON structure, update get_location()
LinkedIn CSV has no Profile URL column LinkedIn changed export format Export "Connections" data type specifically, not the full archive. Check for "URL" or "linkedin_url" column names.
PE/VC match rate too low Missing niche fund names in PEVC_FIRM_NAMES Add firm-specific names relevant to your network (APAC funds, boutique GPs, regional VCs)
False positives (non-PE "Principal" titles) Title pattern too broad Remove r"\bprincipal\b" from PEVC_TITLE_PATTERNS and rely on company-level matching instead
